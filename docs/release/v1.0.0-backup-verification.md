# v1.0.0 — Backup Verification

> No secret values appear in this document. Values are references, not credentials.

## Database backup
- Source: managed Postgres (Lovable Cloud)
- Snapshot cadence: managed automatic daily snapshot
- Pre-deploy snapshot: **required** immediately before the v1.0.0 deploy
- Snapshot timestamp: recorded by the deploy operator at execution time
- Restore owner: platform admin
- Restore instructions: use managed console point-in-time restore to the pre-deploy timestamp
- Expected recovery time (target): under 30 minutes for full restore

## Configuration backup
- Environment variable schema: `src/lib/env-validation/index.ts`
- Feature-flag snapshot: `src/lib/release/v1-manifest.ts::V1_MANIFEST.featureFlags`
- Runtime module registry: `src/lib/runtime-readiness/runtime-evidence.ts`
- Trading-safety flags: enforced false in `V1_MANIFEST.tradingFlags`

## Release artifact backup
- Production build artifact: retained by hosting (previous stable deploy)
- Rollback target: previous stable deployment (see `v1.0.0-rollback.md`)

## Sign-off
- Operator must attach the pre-deploy snapshot timestamp to the launch readiness record before ticking `signoff.human`.
