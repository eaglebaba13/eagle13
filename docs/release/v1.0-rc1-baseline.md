## Known Limitations (RC1)
- Live order execution is disabled at the platform level; RC1 ships strictly as
  a research/analytics product.
- Billing is manual-UPI verification only; no automatic payment provider is
  wired in RC1. Pricing surfaces are labelled as manual verification.
- CoinDCX integration is market-data only.
- AI Market Assistant is deterministic/template-based; no LLM inference in RC1.
- Research Lab and Backtest Lab are research-only; historical performance is
  not a forward guarantee.

## Deferred Beyond RC1
- Automatic payment provider (Razorpay/Stripe) wiring
- Live order execution (kept structurally absent by design)
- Push/mobile-app delivery for Smart Alerts
- Multi-tenant workspace features

## Security Posture (RC1)
- RLS enforced on all public tables; policies co-located with migrations
- Service role key never referenced from client-reachable modules
  (`.server.ts` boundary enforced by TanStack import protection)
- Bearer auth attached client-side via
  `src/integrations/supabase/auth-attacher.ts`; server functions verify via
  `requireSupabaseAuth`
- Admin capabilities gated via `has_role(auth.uid(), 'admin')`
